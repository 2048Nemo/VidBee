# Internationalization (i18n)

Guide for implementing multi-language support.

## Package Structure

```
packages/i18n/src/
├── locales/
│   ├── en.json              # English (base)
│   ├── zh.json              # Chinese
│   ├── zh-TW.json           # Traditional Chinese
│   ├── ja.json              # Japanese
│   ├── ko.json              # Korean
│   └── ...                  # Other languages
├── languages.ts             # Language definitions
├── resources.ts             # Translation resources
├── init.ts                  # i18n initialization
└── index.ts                 # Exports
```

## Language Definition

```typescript
// packages/i18n/src/languages.ts
export const languageList = [
  { code: 'en', name: 'English', nativeName: 'English' },
  { code: 'zh', name: 'Chinese', nativeName: '中文' },
  { code: 'zh-TW', name: 'Traditional Chinese', nativeName: '繁體中文' },
  { code: 'ja', name: 'Japanese', nativeName: '日本語' },
  { code: 'ko', name: 'Korean', nativeName: '한국어' },
] as const

export type LanguageCode = typeof languageList[number]['code']

export const defaultLanguageCode: LanguageCode = 'en'
export const supportedLanguageCodes = languageList.map(l => l.code)
```

## Initialization

```typescript
// packages/i18n/src/init.ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import { defaultLanguageCode, resources } from './resources'

export async function initI18n() {
  await i18n.use(initReactI18next).init({
    lng: defaultLanguageCode,
    fallbackLng: 'en',
    resources,
    interpolation: {
      escapeValue: false,
    },
  })
}

// For Electron main process (no React)
export async function initI18nMain() {
  await i18n.init({
    lng: defaultLanguageCode,
    fallbackLng: 'en',
    resources,
  })
}
```

## Usage in Components

```tsx
// React component
import { useTranslation } from 'react-i18next'

export function SettingsPage() {
  const { t } = useTranslation()

  return (
    <div>
      <h1>{t('settings.title')}</h1>
      <label>{t('settings.language.label')}</label>
    </div>
  )
}
```

## Usage in Main Process

```typescript
// Electron main process
import i18n from 'i18next'
import { initI18nMain } from '@myproject/i18n'

await initI18nMain()

const message = i18n.t('errors.networkError')
```

## Workflow

### 1. Add Translation (English First)

```json
// en.json
{
  "feature": {
    "title": "Feature Title",
    "description": "Description text",
    "actions": {
      "save": "Save",
      "cancel": "Cancel"
    }
  }
}
```

### 2. Use in Code

```tsx
const { t } = useTranslation()
<h1>{t('feature.title')}</h1>
<button>{t('feature.actions.save')}</button>
```

### 3. Check for Missing Translations

```bash
pnpm run check:i18n
```

### 4. Translate to Other Languages

```json
// zh.json
{
  "feature": {
    "title": "功能标题",
    "description": "描述文本",
    "actions": {
      "save": "保存",
      "cancel": "取消"
    }
  }
}
```

## Naming Conventions

- Use dot notation: `namespace.subkey.key`
- Group related keys: `feature.actions.save`
- Keep keys descriptive but concise
- Use consistent naming patterns:
  - `common.*` for shared strings
  - `nav.*` for navigation
  - `errors.*` for error messages
  - `actions.*` for action buttons

## Best Practices

1. **English First** - Always add English translations first
2. **Don't Copy** - Translate properly, don't just copy English
3. **Context** - Use namespace to provide context
4. **Plurals** - Use i18next plural features when needed
5. **Variables** - Use interpolation for dynamic values
   ```json
   { "greeting": "Hello, {{name}}!" }
   ```
   ```tsx
   t('greeting', { name: 'John' })
   ```
